## 🎯 場景 1：請假申請映射

### 定義書中的規則

**Rule 1：額度檢查**

```
定義書中：
────────
IF 請假類型 = "年假" THEN
  檢查員工年度剩餘年假額度
  IF 剩餘額度 ≥ 申請天數 THEN 允許申請
  ELSE 拒絕申請，返回"額度不足"錯誤
END
```

### 代碼中的實現

**reference_implementations/leave_approval_reference.js**

```javascript
// 對應定義書的 Rule 1：額度檢查
function checkLeaveBalance(employeeId, leaveType, dayCount) {
  // 實現定義書中的邏輯

  // 步驟 1：查詢員工年假額度
  const employeeBalance = getEmployeeLeaveBalance(employeeId, leaveType)

  // 步驟 2：比較額度
  if (leaveType === 'vacation') {
    const annualQuota = 15  // 定義書中：年假 15 天
    const usedDays = employeeBalance.used
    const remainingDays = annualQuota - usedDays

    // 步驟 3：做出決策
    if (remainingDays >= dayCount) {
      // 對應：IF 剩餘額度 ≥ 申請天數 THEN 允許申請
      return {
        allowed: true,
        remaining: remainingDays - dayCount,
        message: '年假額度充足，允許申請'
      }
    } else {
      // 對應：ELSE 拒絕申請
      return {
        allowed: false,
        remaining: remainingDays,
        message: `年假額度不足，剩餘 ${remainingDays} 天，無法滿足 ${dayCount} 天申請`
      }
    }
  }
}
```

### 映射對照表

| 定義書 | 代碼實現 | 說明 |
|--------|---------|------|
| `IF 請假類型 = "年假"` | `if (leaveType === 'vacation')` | 檢查假期類型 |
| `檢查員工年度剩餘年假額度` | `getEmployeeLeaveBalance(employeeId, leaveType)` | 查詢額度函數 |
| `剩餘額度 ≥ 申請天數` | `remainingDays >= dayCount` | 條件判斷 |
| `允許申請` | `return { allowed: true, ... }` | 返回批准結果 |
| `拒絕申請，返回"額度不足"` | `return { allowed: false, message: '額度不足' }` | 返回拒絕結果 |

### Rule 2：審批路徑確定

**定義書中：**

```
IF 請假天數 ≤ 2 THEN
  審批人 = 直屬經理
  審批級數 = 1

ELSE IF 請假天數 ≤ 5 THEN
  審批人 = [直屬經理, 部門主管]
  審批級數 = 2

ELSE IF 請假天數 > 5 THEN
  審批人 = [直屬經理, 部門主管, HR 負責人]
  審批級數 = 3

END
```

**代碼中：**

```javascript
// 對應定義書的 Rule 2：審批路徑確定
function determineApprovalPath(dayCount, department) {
  // 實現分級審批邏輯

  if (dayCount <= 2) {
    // 對應：請假天數 ≤ 2
    return {
      approvers: ['DirectManager'],
      levels: 1,
      description: '1 級審批：直屬經理'
    }
  } else if (dayCount <= 5) {
    // 對應：請假天數 ≤ 5
    return {
      approvers: ['DirectManager', 'DepartmentHead'],
      levels: 2,
      description: '2 級審批：經理 + 部門主管'
    }
  } else if (dayCount > 5) {
    // 對應：請假天數 > 5
    return {
      approvers: ['DirectManager', 'DepartmentHead', 'HRDirector'],
      levels: 3,
      description: '3 級審批：經理 + 部門主管 + HR'
    }
  }
}
```

### 修改示例

假設企業修改了定義書，改為：

```
新政策：
- ≤ 3 天：經理批准（改自 2 天）
- ≤ 7 天：經理 + 部門主管（改自 5 天）
- > 7 天：經理 + 部門主管 + HR（改自 5 天）
```

代碼修改如下：

```javascript
// 修改後的代碼
function determineApprovalPath(dayCount, department) {
  if (dayCount <= 3) {  // 改：2 → 3
    return {
      approvers: ['DirectManager'],
      levels: 1
    }
  } else if (dayCount <= 7) {  // 改：5 → 7
    return {
      approvers: ['DirectManager', 'DepartmentHead'],
      levels: 2
    }
  } else if (dayCount > 7) {  // 改：5 → 7
    return {
      approvers: ['DirectManager', 'DepartmentHead', 'HRDirector'],
      levels: 3
    }
  }
}
```

**要點**：只需要改數字，邏輯結構不變！

### Rule 3：自動批准決策

**定義書中：**

```
IF 員工過去12個月內請假記錄 = 優秀（無遲到、無無故缺勤） THEN
  經理直接批准概率 = 95%
  自動批准（無需人工干預）

ELSE IF 員工請假記錄 = 良好 THEN
  經理直接批准概率 = 70%
  需要經理確認

ELSE IF 員工請假記錄 = 一般或較差 THEN
  需要多級審批並確認理由
  自動審批概率 = 30%

END
```

**代碼中：**

```javascript
// 對應定義書的 Rule 3：自動批准決策
function shouldAutoApprove(employeeId, dayCount) {
  // 查詢員工記錄評分
  const employeeRecord = getEmployeeAttendanceRecord(employeeId)

  // 評估員工記錄質量
  if (employeeRecord.rating === 'excellent') {
    // 對應：員工記錄優秀
    if (dayCount <= 2) {
      // 自動批准（對應"自動批准"）
      return {
        autoApprove: true,
        reason: '員工記錄優秀，自動批准',
        confidence: 0.95
      }
    }
  } else if (employeeRecord.rating === 'good') {
    // 對應：員工記錄良好
    return {
      autoApprove: false,
      reason: '員工記錄良好，需要經理確認',
      confidence: 0.70
    }
  } else {
    // 對應：員工記錄一般或較差
    return {
      autoApprove: false,
      reason: '需要多級審批確認',
      confidence: 0.30
    }
  }
}
```

## 🎯 場景 2：報銷申請映射

### Rule 1：收據合規性檢查

**定義書中：**

```
IF 發票金額 ≤ 0 OR 發票金額 > 10,000 THEN
  標記為 "金額異常"，需要人工審查
END

IF 發票日期 > 當前日期 OR 發票日期 < (當前日期 - 90天) THEN
  標記為 "日期異常"，拒絕報銷
END

IF 發票類型 = "普通發票" THEN
  檢查發票號碼格式
  IF 格式正確 THEN 通過
  ELSE 標記為 "發票號無效"，拒絕
END
```

**代碼中：**

```javascript
function validateReceipts(receipts) {
  const validationResults = receipts.map(receipt => {
    // 對應：金額檢查
    if (receipt.amount <= 0 || receipt.amount > 10000) {
      return {
        valid: false,
        issue: '金額異常，需要人工審查',
        receipt_id: receipt.id
      }
    }

    // 對應：日期檢查
    const receiptDate = new Date(receipt.date)
    const today = new Date()
    const daysDiff = Math.floor((today - receiptDate) / (1000 * 60 * 60 * 24))

    if (receiptDate > today || daysDiff > 90) {
      return {
        valid: false,
        issue: '日期異常，超過 90 天報銷期限',
        receipt_id: receipt.id
      }
    }

    // 對應：發票類型和格式檢查
    if (receipt.type === 'normal_invoice') {
      if (!isValidInvoiceNumber(receipt.number)) {
        return {
          valid: false,
          issue: '發票號碼格式無效',
          receipt_id: receipt.id
        }
      }
    }

    // 所有檢查通過
    return {
      valid: true,
      receipt_id: receipt.id
    }
  })

  return validationResults
}
```

### Rule 2：金額和審批層級

**定義書中：**

```
IF 申請金額 ≤ 500 THEN
  審批人 = 直屬經理
  審批級數 = 1

ELSE IF 申請金額 ≤ 2,000 THEN
  審批人 = [直屬經理, 部門主管]
  審批級數 = 2

ELSE IF 申請金額 > 5,000 THEN
  審批人 = [直屬經理, 部門主管, 財務經理, CFO]
  審批級數 = 4

END
```

**代碼中：**

```javascript
function routeExpenseApproval(amount) {
  if (amount <= 500) {
    return {
      approvers: ['DirectManager'],
      levels: 1
    }
  } else if (amount <= 2000) {
    return {
      approvers: ['DirectManager', 'DepartmentHead'],
      levels: 2
    }
  } else if (amount > 5000) {
    return {
      approvers: ['DirectManager', 'DepartmentHead', 'FinanceManager', 'CFO'],
      levels: 4
    }
  }
}
```

**修改示例**：

企業政策改為需要更嚴格的審批：

```
新政策：
- ≤ 300 元：經理（改自 500）
- ≤ 1,500 元：經理 + 部門主管（改自 2,000）
- > 3,000 元：經理 + 部門主管 + 財務（改自 5,000）
```

代碼修改：

```javascript
function routeExpenseApproval(amount) {
  if (amount <= 300) {      // 改：500 → 300
    return { approvers: ['DirectManager'], levels: 1 }
  } else if (amount <= 1500) {  // 改：2000 → 1500
    return { approvers: ['DirectManager', 'DepartmentHead'], levels: 2 }
  } else if (amount > 3000) {   // 改：5000 → 3000
    return {
      approvers: ['DirectManager', 'DepartmentHead', 'FinanceManager'],
      levels: 3
    }
  }
}
```

## 🎯 場景 3：培訓申請映射

### Rule 1：資格檢查

**定義書中：**

```
基礎資格：
  - 員工在公司工作滿 3 個月
  - 過去 6 個月績效評分 ≥ 3.0/5.0
  - 未在試用期內

IF 薪級 ∈ [P1, P2, P3] THEN
  允許的培訓類別 = [通用技能, 行業基礎, 軟技能]
  年度培訓預算 = 2,000 元
END
```

**代碼中：**

```javascript
function checkTrainingEligibility(employeeId) {
  // 查詢員工信息
  const employee = getEmployeeInfo(employeeId)

  // 對應：基礎資格檢查
  const monthsInCompany = getMonthsInCompany(employeeId)
  if (monthsInCompany < 3) {
    return {
      eligible: false,
      reason: '員工入職不滿 3 個月'
    }
  }

  const performanceScore = getPerformanceScore(employeeId)
  if (performanceScore < 3.0) {
    return {
      eligible: false,
      reason: '過去 6 個月績效評分低於 3.0'
    }
  }

  // 對應：薪級限制
  const allowedCategories = {
    'P1': ['general_skills', 'industry_basics', 'soft_skills'],
    'P2': ['general_skills', 'industry_basics', 'soft_skills'],
    'P3': ['general_skills', 'industry_basics', 'soft_skills'],
    'P4': ['general_skills', 'professional_skills', 'leadership_basics'],
    'P5': ['all']
  }

  const annualBudget = {
    'P1': 2000,  // 對應定義書：P1-P3 級 2000 元
    'P2': 2000,
    'P3': 2000,
    'P4': 5000,
    'P5': 10000
  }

  return {
    eligible: true,
    allowedCategories: allowedCategories[employee.level],
    annualBudget: annualBudget[employee.level]
  }
}
```

## 📝 通用映射模式

### 模式 1：IF-ELSE 條件

**定義書**：
```
IF 條件1 THEN
  動作A
ELSE IF 條件2 THEN
  動作B
ELSE
  動作C
END
```

**代碼**：
```javascript
if (condition1) {
  // 動作A
  return resultA
} else if (condition2) {
  // 動作B
  return resultB
} else {
  // 動作C
  return resultC
}
```

### 模式 2：表格決策

**定義書**：
```
| 金額 | 審批人 | 級數 |
|-----|--------|------|
| ≤ 500 | 經理 | 1 |
| ≤ 2000 | 經理 + 主管 | 2 |
| > 2000 | 經理 + 主管 + CFO | 3 |
```

**代碼**：
```javascript
const APPROVAL_MATRIX = {
  500: { approvers: ['Manager'], levels: 1 },
  2000: { approvers: ['Manager', 'Head'], levels: 2 },
  Infinity: { approvers: ['Manager', 'Head', 'CFO'], levels: 3 }
}

function getApprovalPath(amount) {
  for (const [limit, config] of Object.entries(APPROVAL_MATRIX)) {
    if (amount <= limit) return config
  }
}
```

### 模式 3：計算和驗證

**定義書**：
```
計算 = 年度預算 - 已用金額
IF 計算結果 ≥ 申請金額 THEN
  批准
ELSE
  拒絕
END
```

**代碼**：
```javascript
const remaining = annualBudget - usedAmount
if (remaining >= requestAmount) {
  return { approved: true, remaining: remaining - requestAmount }
} else {
  return { approved: false, deficit: requestAmount - remaining }
}
```

## 🔄 改進定義書時的代碼修改

### 步驟 1：識別變更

```
定義書中的變更：
  Rule 2：審批層級從 2 層改為 3 層
  Rule 3：自動批准的條件從「優秀」改為「優秀或良好」
```

### 步驟 2：找到對應代碼

```javascript
// 對應 Rule 2 的代碼位置
function determineApprovalPath(dayCount) { ... }

// 對應 Rule 3 的代碼位置
function shouldAutoApprove(employeeRecord) { ... }
```

### 步驟 3：進行修改

```javascript
// 修改前
if (dayCount <= 2) {
  return { approvers: ['DirectManager'], levels: 1 }
}

// 修改後（新增 HR 層級）
if (dayCount <= 2) {
  return {
    approvers: ['DirectManager'],
    levels: 1
  }
}
// ... 其他條件
```

### 步驟 4：測試修改

```javascript
// 測試用例
const testCases = [
  { dayCount: 1, expected: 1 },
  { dayCount: 2, expected: 1 },
  { dayCount: 3, expected: 2 },
  { dayCount: 6, expected: 3 }
]

testCases.forEach(test => {
  const result = determineApprovalPath(test.dayCount)
  assert(result.levels === test.expected)
})
```
