| 任務類型 | 推薦模型 | 預算 | 說明 |
|---------|--------|------|------|
| 快速檢查 | Haiku | 0.05-0.1 | 語法、格式檢查 |
| 標準審查 | Sonnet | 0.2-0.4 | 邏輯、設計審查 |
| 複雜分析 | Opus | 0.5+ | 架構設計、高級優化 |

#### 實踐 3：四階段驗證流程

```
Phase 1: 語法檢查 (快速發現表面問題)
        ↓
Phase 2: 邏輯審查 (深度分析業務邏輯)
        ↓
Phase 3: Bug 修復 (生成修復方案)
        ↓
Phase 4: 測試生成 (確保質量覆蓋)
```

#### 實踐 4：預算控制方程式

```
總預算 = 各 Phase 預算之和 + 應急預算

成本效益 = (完成工作量) / (消耗單位)

預留率 = (剩餘預算) / (總預算) >= 60%
```
