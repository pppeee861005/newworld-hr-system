### A.5 成本優化策略

#### 策略 1：任務分解
將大型任務分解為小任務，使用更便宜的模型：

```
❌ 不好：用 Opus 生成完整的 500 行 Workflow（1.0 單位）
✅ 好：
  1. 用 Sonnet 生成框架（0.2 單位）
  2. 用 Haiku 生成工具函數（0.05 單位）
  3. 用 Sonnet 整合和優化（0.15 單位）
  總計：0.4 單位（節省 60%）
```

#### 策略 2：迭代優化
先用便宜模型快速原型，再用高級模型優化：

```
❌ 不好：直接用 Opus 生成完美代碼（0.8 單位）
✅ 好：
  1. Haiku 生成初版（0.05 單位）
  2. Sonnet 優化邏輯（0.15 單位）
  3. Opus 審查關鍵部分（0.2 單位）
  總計：0.4 單位（節省 50%）
```

#### 策略 3：批量處理
合併相似任務，減少 API 調用次數：

```
❌ 不好：為每個配置文件單獨調用（3 次 × 0.02 = 0.06 單位）
✅ 好：一次生成所有配置文件（1 次 × 0.05 = 0.05 單位）
節省：17%
```

#### 策略 4：緩存復用
保存通用代碼模板，減少重複生成：

```
示例：Workflow 範本
首次生成：Sonnet 0.3 單位
後續使用：直接複製 + Haiku 微調 0.05 單位
節省：83%（第 2 次起）
```
